Проект запускается в докере командой из корня проекта:
`make start_project`

Открывается в браузере по ссылке:
http://localhost:8080/

Базовый юзер:
Логин - user@example.com
Пароль - user@example.com

Запуск тестов:
- `cd frontend`
- `npm run test:unit`

## Дополнительные функции сверх технического задания

### Авторизация и безопасность

1. **Регистрация пользователей** - реализована полная система регистрации с формой на странице `/signup`, включающая валидацию всех полей (имя, email, телефон, пароль)

### Избранное

2. **Избранные пиццы** - реализована возможность добавлять пиццы в избранное (до того, как положили в корзину).
Список избранного хранится на странице /favorites (есть пункт меню в хедере и профиле)
3. **Добавление пиццы из избранного** - пиццу из избранного можно добавить в конструктор и опять добавить в заказ
4. **Удаление пиццы из избранного** - пиццу из избранного можно удалить

### Анимации и визуальные эффекты

5. **Анимация "полет к корзине"** - при добавлении пиццы в корзину запускается анимация полета элемента от кнопки до иконки корзины в шапке
6. **Расширенные анимации списков** - использование `TransitionGroup` для плавных анимаций при добавлении/удалении элементов в корзине и списке заказов
7. **Анимации переходов между страницами** - плавные переходы при навигации по приложению
8. **Добавлены индикаторы загрузки**

### Удобство использования (UX)

9. **Сохранение корзины в localStorage** - корзина автоматически сохраняется в локальном хранилище браузера и восстанавливается при перезагрузке страницы
10. **Фильтрация заказов** - можно искать заказ по имени пиццы в списке заказов
11. **Отображение аватара пользователя** - аватар пользователя отображается в шапке сайта и на странице профиля (для скорости не делала загрузку фото, поэтому аватар только для базового пользователя user@example.com / user@example.com)
12. **Предзаполнение телефона** - поле телефона на странице корзины автоматически заполняется номером из профиля авторизованного пользователя
13. **Редирект после авторизации** - после успешной авторизации пользователь автоматически возвращается на страницу, с которой был перенаправлен на форму входа
14. **Кнопка "Отмена" в форме адреса** - возможность отменить редактирование адреса без сохранения изменений
15. **Подтверждение удаления заказа** - диалог подтверждения перед удалением заказа для предотвращения случайных действий
16. **Валидация перед оформлением заказа** - проверка заполнения обязательных полей (телефон, адрес) с понятными сообщениями об ошибках

### Технические улучшения
