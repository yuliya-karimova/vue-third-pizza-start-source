Проект запускается в докере командой из корня проекта:
`make start_project`

Открывается в браузере по ссылке:
http://localhost:8080/

Запуск тестов:
- `cd frontend`
- `npm run test:unit`

## Дополнительные функции, реализованные сверх технического задания

В проекте реализован ряд дополнительных функций и улучшений, которые не были описаны в техническом задании, но улучшают пользовательский опыт и качество приложения:

### Авторизация и безопасность

1. **Регистрация пользователей** - реализована полная система регистрации (`signUp`), позволяющая новым пользователям создавать аккаунты через API
2. **Выход из системы** - кнопка "Выйти" в шапке сайта с корректной очисткой сессии
3. **Защита от повторных запросов** - реализована защита от множественных одновременных вызовов API при загрузке пользователя

### Работа с данными

4. **Сохранение корзины в localStorage** - корзина автоматически сохраняется в локальном хранилище браузера и восстанавливается при перезагрузке страницы
5. **Сортировка заказов** - заказы автоматически сортируются по ID (новые заказы отображаются первыми)
6. **Форматирование чисел** - цены и суммы отображаются с разделителями тысяч в российском формате (например: "1 234 ₽")

### Анимации и визуальные эффекты

7. **Анимация "полет к корзине"** - при добавлении пиццы в корзину запускается анимация полета элемента от кнопки до иконки корзины в шапкеT
8. **Расширенные анимации списков** - использование `TransitionGroup` для плавных анимаций при добавлении/удалении элементов в корзине и списке заказов
9. **Анимации переходов между страницами** - плавные переходы при навигации по приложению

### Удобство использования (UX)

10. **Предзаполнение телефона** - поле телефона на странице корзины автоматически заполняется номером из профиля авторизованного пользователя
11. **Редирект после авторизации** - после успешной авторизации пользователь автоматически возвращается на страницу, с которой был перенаправлен на форму входа
12. **Автоматическое открытие формы адреса** - если у пользователя нет сохраненных адресов, форма создания нового адреса открывается автоматически
13. **Кнопка "Отмена" в форме адреса** - возможность отменить редактирование адреса без сохранения изменений

### Функциональность корзины

14. **Удаление пиццы из корзины** - дополнительная кнопка "Удалить" для быстрого удаления пиццы из корзины (кроме удаления через уменьшение количества до 0)
15. **Улучшенная валидация форм** - валидация полей в реальном времени с отображением ошибок при потере фокуса (onBlur)
16. **Валидация перед оформлением заказа** - проверка заполнения обязательных полей (телефон, адрес) с понятными сообщениями об ошибках

### Отображение данных

17. **Подтверждение удаления заказа** - диалог подтверждения перед удалением заказа для предотвращения случайных действий
18. **Отображение аватара пользователя** - аватар пользователя отображается в шапке сайта и на странице профиля
19. **Отображение количества ингредиентов** - в списках заказов и корзине показывается количество одинаковых ингредиентов (например: "моцарелла ×2")
20. **Обработка ошибок с понятными сообщениями** - улучшенная обработка ошибок API с извлечением и отображением понятных сообщений пользователю из различных форматов ответов сервера

### Технические улучшения

21. **Дополнительный маршрут `/user`** - создан алиас для страницы профиля (`/user` → `/profile`)
22. **Реактивное отслеживание изменений** - использование Vue `watch` для автоматического обновления данных при изменении состояния пользователя
23. **Исправлена работа модальных окон** - добавлен компонент `ModalDialog` в `FavoritesView.vue` для корректной работы кнопки "Удалить" в избранном, исправлена проблема с типами Ref в `OrdersView.vue`
24. **Оптимизация стилей** - проведена оптимизация SCSS структуры:
    - Удалены дублирующиеся стили (`.close` был в `popup.scss` и `blocks/close.scss`)
    - Централизованы общие стили блоков в `common.scss` (title, button, input, select, product, counter, close)
    - Убраны повторяющиеся импорты из компонентов Vue
    - Удалены неиспользуемые файлы стилей (`main.scss`)
25. **Улучшена работа с полем телефона**:
    - Исправлена проблема отображения "[object Object]" в инпуте телефона
    - Обновлен формат телефона на российский стандарт: `+7 999 777-77-77`
    - Улучшена валидация телефона с проверкой на точное количество цифр (11 цифр)
    - Обновлена функция нормализации для корректной отправки на сервер

---

Эти дополнительные функции делают приложение более удобным, надежным и приятным в использовании, не нарушая требований основного технического задания.
